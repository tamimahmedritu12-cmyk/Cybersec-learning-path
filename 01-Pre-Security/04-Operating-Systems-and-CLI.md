## Key Technical Concepts

### 1. Operating System Fundamentals
An Operating System (OS) acts as an intermediary between the user/applications and the computer hardware. It manages resources, file systems, and hardware components (CPU, RAM, Storage).

### 2. Linux Command Line Interface (CLI) Basics
Linux is heavily used in cybersecurity infrastructure. Navigating through the terminal efficiently is a core requirement.
* `ls` / `ls -la`: Lists files, including hidden ones with permissions.
* `cd`: Changes directories.
* `cat`: Displays file contents.
* `grep`: Searches for specific patterns inside text files or command outputs (e.g., parsing logs).
* `chmod` / `chown`: Modifies file permissions and ownership.

### 3. Windows Desktop & CLI Essentials
Windows system administration requires understanding built-in management utilities and the Command Prompt/PowerShell.
* **Key GUI Tools:** Task Manager, Event Viewer (crucial for defender logs), Services, Registry Editor.
* **CLI Basics:**
  * `dir`: Lists directory contents.
  * `type`: Displays contents of a file (similar to Linux `cat`).
  * `whoami`: Displays current logged-in user context.
  * `ipconfig`: Displays network configuration details.

---

## 🛠️ Key Takeaway for Cybersecurity
Attackers and defenders both rely heavily on the command line. Knowing how to navigate directories, view hidden files, and identify system configurations via CLI is essential for live-incident response and privilege escalation phases.
