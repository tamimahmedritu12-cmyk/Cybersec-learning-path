## Key Technical Concepts

### 1. What is Cloud Computing?
Cloud computing is the on-demand delivery of IT resources (compute, storage, databases) over the internet with pay-as-you-go pricing.

### 2. Deployment Models
* **Public Cloud:** Resources owned and operated by a third-party provider (AWS, Azure, Google Cloud).
* **Private Cloud:** Infrastructure used exclusively by a single business or organization.
* **Hybrid Cloud:** Combines public and private clouds, allowing data and applications to be shared between them.

### 3. Service Models (The Cloud Pyramid)
* **IaaS (Infrastructure as a Service):** Renting raw servers, storage, and networking (e.g., AWS EC2). You manage the OS, runtime, and apps.
* **PaaS (Platform as a Service):** Provides a platform for developers to build apps without worrying about underlying servers (e.g., Heroku, AWS Elastic Beanstalk).
* **SaaS (Software as a Service):** End-user software delivered over the web (e.g., Google Drive, Microsoft 365).

## Key Takeaway for Cybersecurity
As modern enterprises migrate fully to the cloud, cyber warfare increasingly shifts towards attacking cloud configurations. Understanding the Shared Responsibility Model (what the provider secures vs. what the customer must secure) is vital for cloud security engineering.
